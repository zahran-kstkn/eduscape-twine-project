# 🚨 Cyber Escape Room - Project Game Edukasi Twine

Halo! Ini adalah repository untuk project game edukasi *cybersecurity escape room* (Choose Your Own Adventure) yang lagi kita bikin pakai Twine. Game ini disetting untuk mensimulasikan skenario serangan siber/scam di dunia nyata dengan visual ala layar smartphone.

Project ini juga sudah otomatis live di GitHub Pages! Kamu bisa coba mainkan versi terbarunya di sini:
👉 *[MASUKKAN LINK GITHUB PAGES KAMU DI SINI]*

## ⚠️ ATURAN UTAMA KOLABORASI (Sistem Estafet)
Karena Twine menyimpan seluruh data game dalam **satu file tunggal (`index.html`)**, kita **TIDAK BISA** mengedit file ini secara bersamaan di waktu yang sama. Jika dipaksa, filenya akan bentrok (*merge conflict*) atau kerjaan salah satu dari kita bakal tertimpa dan hilang.

Jadi, kita wajib pakai sistem **"Oper Tongkat Estafet"**:

1. **Kabari di chat** sebelum mulai ngedit ("Eh, gw mau update Twine-nya ya").
2. Jika situasi aman, **Download file `index.html` terbaru** dari repo GitHub ini.
3. **Import** file tersebut ke aplikasi Twine kamu, lalu silakan edit/tambah ceritanya.
4. Selesai ngedit, pilih **Publish to File** di Twine, lalu pastikan nama filenya tetap `index.html`.
5. **Upload kembali ke GitHub** (Commit changes) untuk memperbarui versi live.
6. **Kabari lagi di chat** kalau kamu sudah selesai ("Udah gw upload ya, silakan kalau mau gantian").

---

## 🛠️ Cara Memasukkan File ke Twine
Jika kamu bingung cara membuka file project ini di Twine:
1. Buka aplikasi Twine (atau lewat browser di [twinery.org](https://twinery.org/)).
2. Di halaman awal dashboard Twine, klik menu **Import** (atau **Import From File**).
3. Pilih file `index.html` yang baru saja kamu download dari GitHub.
4. *Tips:* Kalau di dashboard Twine kamu masih ada file versi lama, hapus dulu versi lama tersebut agar tidak tertukar saat mengedit.
